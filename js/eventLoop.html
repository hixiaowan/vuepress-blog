    微任务、宏任务与Event-Loop

    三人所为 2022/3/25

    # JavaScript 执行机制

    • 执行栈（call stack）
      • 同步
        • 主线程
      • 异步
        • Event Table（事件列表）
          • 注册回调函数
          • Event Queue（事件队列）
            • 微任务队列
              • process.nextTick（node 独有），Promise，async/await
            • 宏任务队列
              • setTimeout，setInterval，script（整体代码）， DOM 事件，AJAX 请求，requestAnimationFrame

    # Event Loop（事件循环）

    • 浏览器的事件循环由一个宏任务队列+多个微任务队列组成
    • 每轮循环都是由一个宏任务+一个微任务队列（所有在此期间入队的微任务）组成。

    # 常见问题

    • 如何知道主线程执行栈为空？
      • js 引擎存在 monitoring process（监测过程）进程，会持续不断的检查主线程执行栈是否为空
    • 为什么会分为宏任务和微任务？
      • 事件循环由宏任务和在执行宏任务期间产生的所有微任务组成。完成当下的宏任务后，会立刻执行所有在此期间入队的微任务。
      • 这种设计是为了给紧急任务一个插队的机会，否则新入队的任务永远被放在队尾。区分了微任务和宏任务后，本轮循环中的微任务实际上就是在插队，这样微任务中所做的状态修改，在下一轮事件循环中也能得到同步。

    # 注意事项

    • 切记 script(整体代码)是一个宏任务
    • 事件循环中的任务被分为宏任务和微任务，是为了给高优先级任务一个插队的机会：==微任务比宏任务有更高优先级==。

    # 延伸（Node 环境中的事件循环）

    • Node 环境中的事件循环与浏览器事件循环的区别
      • 宏任务与微任务都存在优先级排序
      • 每轮循环都是由一整个宏任务队列+一个微任务队列。
      • node11 之后，node 和浏览器的规律趋同，但优先级排序依旧保留
